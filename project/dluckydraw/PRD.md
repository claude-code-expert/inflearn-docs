# PRD — Lucky Draw 룰렛 추첨 사이트

> **이 문서를 받은 Claude Code의 작업 범위**
> 단일 HTML 파일(`lucky-draw.html`)에 HTML + CSS + JavaScript를 모두 포함한 정적 럭키 드로우 사이트를 구현합니다. 외부 빌드 도구, 프레임워크, 패키지 매니저 모두 사용하지 않습니다. 함께 제공되는 `TRD.md`와 `REQUIREMENTS.md`를 반드시 함께 읽고 작업을 시작하세요.

---

## 1. 제품 개요

### 1.1 한 줄 정의
경품 추첨이 필요한 모든 상황에서, 화면 한 장으로 입력·추첨·결과 확인까지 끝낼 수 있는 **회전 룰렛 기반 럭키 드로우 웹 사이트**.

### 1.2 배경
오프라인 행사·온라인 세미나·팀 회식 자리에서 즉석으로 경품을 추첨할 때, 대부분의 사람들은 종이 제비뽑기·엑셀 random 함수·외부 추첨 사이트를 사용한다. 이는 다음과 같은 문제가 있다.

- **신뢰성 부족** — 누군가가 결과를 조작할 여지가 있는 것처럼 보임
- **시각적 재미 부재** — 단순 결과 출력은 행사 분위기를 끌어올리지 못함
- **이동 비용** — 외부 추첨 사이트는 광고·회원가입·과한 장식이 많음

회전 룰렛은 **돌아가는 과정 자체가 콘텐츠**이고, 사용자가 결과를 눈으로 따라가며 검증할 수 있어 신뢰감을 준다.

### 1.3 목표
- 첫 방문자가 **30초 안에** 경품을 입력하고 첫 추첨을 완료할 수 있다
- 한 화면에서 **입력 → 추첨 → 결과 → 다음 추첨**의 사이클이 끊김 없이 돌아간다
- 결과가 **눈으로 확인 가능한 방식**(휠이 멈추는 위치 = 당첨)으로 표시되어 신뢰감을 준다

## 2. 대상 사용자

| 사용자 유형 | 사용 맥락 | 핵심 니즈 |
|---|---|---|
| 사내 행사 진행자 | 분기 회의·송년회·팀 빌딩 | 빠른 세팅, 큰 화면 표출, 다회 추첨 |
| 온라인 세미나 호스트 | 화상회의 화면 공유 | 시각적으로 명확한 결과, 사운드 불필요 |
| 매장 운영자 | 고객 이벤트, SNS 라이브 | 신뢰감 있는 결과, 모바일·태블릿 대응 |
| 학생·일반 사용자 | 친구 모임, 게임 | 무료, 가입 불필요, 즉시 사용 |

**핵심 페르소나**: "행사 시작 5분 전에 경품 추첨 도구가 필요해진 진행자"

---

## 3. 핵심 사용 시나리오

### 시나리오 A — 즉석 추첨 (가장 빈번)
1. 행사 진행자가 URL을 열거나 HTML 파일을 더블클릭한다
2. 상품명("스타벅스 쿠폰")과 갯수(1)를 입력하고 추가 버튼을 누른다
3. 같은 방식으로 2~5개 경품을 더 등록한다
4. 중앙의 SPIN 버튼을 누른다 → 휠이 5~6초간 회전 후 정지
5. 당첨 결과가 화면 중앙에 큰 글자로 표시된다
6. 진행자가 당첨자에게 경품을 전달한 뒤, 다시 SPIN을 눌러 다음 추첨을 진행한다

### 시나리오 B — 빠른 체험 (첫 방문)
1. 사용자가 사이트에 처음 방문한다
2. "예시 불러오기" 버튼을 눌러 샘플 경품 5종을 즉시 채운다
3. SPIN을 눌러 동작을 확인한다

### 시나리오 C — 다회 연속 추첨
1. 진행자가 "스타벅스 쿠폰 5개", "치킨 기프티콘 3개", "꽝 10개"를 등록한다
2. SPIN을 연속 10회 누른다 → 매번 당첨된 경품의 갯수가 1씩 차감된다
3. 갯수가 0이 된 경품은 휠에서 자동 제거되고, 남은 경품으로만 다시 추첨된다
4. 모든 경품이 소진되면 휠이 빈 상태로 표시되고 SPIN 버튼이 비활성화된다

## 4. 핵심 가치 제안

1. **단일 파일 / 제로 백엔드** — 인터넷 없이도 동작하는 정적 페이지
2. **한 화면 완결** — 입력·추첨·결과·기록이 한 페이지에 모두 보임
3. **시각적 신뢰성** — 휠이 돌아가는 모습이 그대로 결과의 근거가 됨
4. **가중치 추첨** — 갯수가 많은 경품일수록 휠에서 더 넓은 영역을 차지하여 자연스러운 확률 가중치 적용

## 6. 범위 (Scope)

### 6.1 In-Scope (이번 버전에 포함)
- 경품 추가·삭제·갯수 입력
- 회전 룰렛 휠 표시 및 추첨 애니메이션
- 당첨 결과 표시 (휠 위 표시 + 모달 + 결과 패널)
- 당첨 시 시각 효과 (컨페티)
- 당첨 기록 누적 표시
- 통계 (남은 경품 수 / 추첨한 횟수)
- 전체 초기화, 예시 불러오기 보조 기능

### 6.2 Out-of-Scope (이번 버전에서 제외)
- 사용자 계정·로그인
- 추첨 결과 서버 저장·공유 링크 생성
- 음향 효과 (사운드)
- 다국어 (영어·일본어)
- 광고·추적 코드
- 다크 모드 토글 (단일 테마로 충분)
- 휠 외 추첨 방식 (룰렛, 카드 뽑기, 가챠 등)
- 추첨 결과 조작 방지를 위한 서명·해시 검증
- 로컬 스토리지 영속화 (새로고침 시 초기 상태로 시작)

## 7. 디자인 원칙 (AI 흔적 배제)

`REQUIREMENTS.md`의 UI/UX 가이드라인에서 상세 규칙을 정의하지만, **PRD 차원에서 다음 원칙을 합의**한다.

1. **억제된 색상 팔레트** — 화려한 다색 배합을 피하고, 명확한 2~3색 + 단일 액센트로 구성
2. **물리적 사물의 인상** — 디지털 그라데이션이 아니라, 실제 룰렛 휠처럼 평면적인 색면 분할
3. **장식의 절제** — backdrop blur, 다중 그림자, 별빛 배경 등 모든 "분위기 잡기" 효과 금지
4. **타이포그래피 일관성** — 본문 폰트(Pretendard 또는 시스템 sans-serif) 하나로 통일, 디스플레이 폰트 금지
5. **모서리·테두리의 절제** — 둥근 모서리는 4~8px 한 가지 값, 테두리는 1px 또는 사용 안 함

## 8. 마일스톤

| 단계 | 산출물 | 검증 기준 |
|---|---|---|
| M1: 기본 UI | 경품 입력 폼 + 빈 휠 표시 | 입력한 경품이 리스트에 표시됨 |
| M2: 휠 렌더링 | 등록된 경품이 휠에 색상별 슬라이스로 분할 표시 | 갯수에 비례하여 슬라이스 크기 결정 |
| M3: 회전 애니메이션 | SPIN 버튼 → 휠 회전 → 정지 | 5~6초간 ease-out 감속, 포인터 아래에 정지 |
| M4: 결과 표시 | 당첨 결과 + 모달 + 컨페티 | 정지한 슬라이스 = 당첨 경품 일치 |
| M5: 다회 추첨 | 갯수 자동 차감, 0이 되면 휠에서 제거 | 모든 경품 소진 시 SPIN 비활성화 |
| M6: 마무리 | 기록·통계·초기화·예시 | `REQUIREMENTS.md` 체크리스트 100% 통과 |

## 9. 위험 요소 및 대응

| 위험 | 영향 | 대응 |
|---|---|---|
| 휠이 멈춘 위치와 결과 표시가 일치하지 않으면 신뢰성 파괴 | 치명적 | TRD의 각도 계산 알고리즘을 정확히 따를 것. 시각적 검증 필수 |
| 경품이 많아질수록 슬라이스가 좁아져 라벨이 깨짐 | 중 | 슬라이스 각도에 따라 라벨 truncate, 일정 각도 미만이면 생략 |
| 모바일 화면에서 휠이 잘리거나 작아짐 | 중 | aspect-ratio 1:1 유지, 반응형 레이아웃 |
| 컨페티 애니메이션이 저사양 기기에서 끊김 | 하 | 파티클 수 제한, 일정 시간 후 자동 정리 |
