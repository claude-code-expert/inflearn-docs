# REQUIREMENTS — Lucky Draw 기능 명세 및 검증

> **역할**: 구현이 만족해야 할 **기능 명세 · 상호작용 · UI 가이드 · 검증 체크리스트**.
> 데이터 모델·알고리즘·상수는 `TRD.md`를 참조하고 여기서는 반복하지 않는다. (`PRD.md` 목적, `ANALYSIS.md` 결정·이력)
> 작업 순서: PRD → TRD → REQUIREMENTS 를 읽고, 구현 후 §6 체크리스트를 한 줄씩 검증한다.

## 1. 기능 일람

| ID | 기능 | 우선순위 | 비고 |
|---|---|---|---|
| F1 | 인원수 입력 | MUST | 번호 풀 1~N (1~999) |
| F2 | 경품 추가 | MUST | 상품명·갯수 입력 |
| F3 | 경품 삭제 | MUST | 항목별 × 버튼 |
| F4 | 동일 이름 경품 자동 병합 | MUST | 갯수만 증가 |
| F5 | 원판 자동 렌더 (고정 섹션 + 꽝) | MUST | 경품 변경 시 즉시 반영 |
| F6 | 번호 추첨 (즉시 공개) | MUST | 균등 랜덤, 중복 없음, ~0.8초 |
| F7 | 경품 원판 회전 | MUST | 번호 공개 후, 5~6초 ease-out |
| F8 | 당첨 결과 3중 표시 | MUST | 결과 패널 + 모달 + 기록 |
| F9 | 당첨 시 컨페티 | MUST | Canvas, 당첨만(꽝 제외) |
| F10 | 재고 차감 + 재배정 | MUST | 소진 칸을 다른 경품으로(없으면 꽝), 섹션 수 불변 |
| F11 | 당첨 기록 누적 | MUST | 번호 + 상품명 + 시간 |
| F12 | 통계 | MUST | 남은 경품 / 추첨 / 남은 인원 |
| F13 | 전체 초기화 | MUST | 확인 다이얼로그 |
| F14 | 예시 불러오기 | SHOULD | 첫 사용자 온보딩 |

## 2. 사용자 스토리 + 수용 기준

### F1. 인원수 입력
- [ ] `1~999` 정수만, 999 초과 시 999로 제한.
- [ ] 통계 "남은 인원"이 `인원수 − 뽑은 번호 수`로 실시간 갱신.

### F2~F4. 경품 추가/삭제/병합
- [ ] 상품명이 비어있으면 추가 안 됨 + 상품명 입력란 shake.
- [ ] 갯수가 1 미만/빈값/비정수면 추가 안 됨 + 갯수 입력란 shake.
- [ ] 갯수 999 초과 시 999로 제한. 상품명 1~20자(`maxlength="20"`).
- [ ] 추가 성공 시 입력란 초기화 + 상품명 포커스. 상품명·갯수 input에서 **Enter = 추가**.
- [ ] 같은 이름 재추가 → 리스트 한 줄, 갯수만 증가.
- [ ] × 버튼으로 즉시 삭제(확인 없음), 삭제 후 원판·통계 즉시 갱신.
- [ ] `count`가 0인 경품도 리스트에는 **회색으로 표시**.

### F5. 원판 자동 렌더 (TRD §5.1·§5.2)
- [ ] 경품 0개 → 원판 중앙 "경품을 추가해주세요".
- [ ] 경품 등록 → **경품 갯수 합 + 꽝 1칸**의 고정 섹션이 2색 교차(흑/백)로 렌더, 꽝은 회색.
- [ ] 슬라이스 안 텍스트 라벨(색맹 대응). `S≥12`면 truncate, 각도 10° 미만이면 라벨 생략.

### F6. 번호 추첨 (TRD §5.3)
- [ ] SPIN 시 `1~N` 미추첨 번호 중 균등 랜덤 1개를 ~0.8초 만에 공개("당첨 번호 N번").
- [ ] **같은 번호는 다시 뽑히지 않는다(1인 1회).** 모든 번호 소진 시 더 추첨되지 않음.

### F7. 경품 원판 회전 (TRD §5.4) ⭐
- [ ] 번호 공개 **후** 원판이 5.5초간 회전(최소 6바퀴, ease-out, 관성 느낌).
- [ ] 회전 중 SPIN·입력·추가·삭제·초기화·예시 비활성(중복 방지).
- [ ] **멈춘 슬라이스 = 결과** (12시 포인터 아래). 슬라이스 한가운데서 약간 어긋남(±35% jitter).

### F8. 결과 3중 표시
- [ ] 결과 패널(`#번호 → 상품명`) · 모달(번호 + 상품명, 당첨 시 이모지 1개) · 기록에 **동시 표시**.
- [ ] 모달은 닫기 버튼 · 배경 클릭 · Esc로 닫힘.

### F9. 컨페티 (TRD §5.6)
- [ ] 당첨(경품) 시에만 발사, 꽝은 미발사. 중앙 + 0.2/0.4초 좌·우. 화면 밖 자동 제거. `prefers-reduced-motion`이면 생략.

### F10. 재고 차감 + 재배정 (TRD §5.5)
- [ ] 당첨된 경품 `count` 1 감소. 0이 되면 그 칸을 **재고 최다 경품**으로 재배정, 없으면 꽝.
- [ ] 재배정 후에도 **섹션 수는 변하지 않는다.**

### F11. 당첨 기록
- [ ] 매 추첨 후 `{번호, 상품명, 시간(HH:MM:SS)}` 추가, `#일련번호`, **최신이 위**.

### F12. 통계
- [ ] 남은 경품 = 모든 `count` 합, 추첨 = 뽑은 번호 수, 남은 인원 = `N − 추첨`. 실시간 갱신.

### F13. 전체 초기화
- [ ] `confirm()` 확인 → 경품·기록·번호·인원·회전 모두 리셋. 원판은 **회전 없이 즉시 0도**, 결과 "READY TO SPIN".

### F14. 예시 불러오기
- [ ] 경품이 이미 있으면 확인 다이얼로그. 예시: **스타벅스 쿠폰 1 · 치킨 기프티콘 2 · 편의점 5천원 3 · 문상 1만원 2 + 인원 10**. (꽝은 자동 1칸이므로 경품 항목에 넣지 않는다.)

## 3. 상호작용 명세

### 3.1 키보드 흐름
Tab: 인원수 → 상품명 → 갯수 → 추가 → 경품 × 버튼들 → 예시/초기화 → SPIN. 각 input에서 **Enter = 추가**. 모달 열림 시 Esc=닫기.

### 3.2 입력 검증
| 필드 | 규칙 | 위반 시 |
|---|---|---|
| 인원수 | 1~999 정수 | 999 초과는 999로 제한 |
| 상품명 | 1~20자, 비어있지 않음 | shake + 포커스, 추가 막음 |
| 갯수 | 1~999 정수 | shake + 포커스, 추가 막음(999 초과는 제한) |

### 3.3 상태별 SPIN 버튼
| 상태 | SPIN |
|---|---|
| 인원수 0 또는 경품 0 | 비활성 |
| 준비 완료 (번호 남음) | 활성 |
| 회전 중 | 비활성 |
| 모든 번호 소진 | 비활성 |

## 4. UI/UX 디자인 가이드 (AI 흔적 제거 — 핵심)

> 색·간격·라디우스의 **구체 값은 `TRD.md` §6 토큰**을 따른다. 여기서는 지켜야 할 원칙과 금지 항목을 정의한다.

### 4.1 배제 항목 (절대 금지)
| 카테고리 | 금지 |
|---|---|
| 그라데이션 | 배경 `linear/radial-gradient`, 그라데이션 텍스트(`-webkit-background-clip:text`) |
| Glassmorphism | `backdrop-filter: blur`, 반투명 카드 |
| 다색 팔레트 | 무지개·파스텔·네온 슬라이스/컨페티 |
| 다중 그림자 | `box-shadow` 2겹 이상, 색 있는 그림자 |
| 분위기 효과 | 별빛/입자 배경, 호버 시 `translateY` lift |
| 디스플레이 폰트 | Bungee·Pacifico 등 |
| 이모지 남용 | 페이지 전체 2~3개 초과 |
| 다크모드 기본 | 검정 배경 + 네온 |

### 4.2 권장
- **색**: 흑/백 + **단일 액센트(빨강)**. 보조색 없음(회색 명도로 계층). 액센트는 **포인터·SPIN에만**.
- **원판**: 짝/홀 인덱스 검정/백색 2색 교차, **꽝은 중립 회색**. 식별은 슬라이스 라벨 텍스트로.
- **타이포**: 시스템 sans 하나, 굵기 400/700/800.
- **모서리·그림자**: `border-radius:6px`(원형 제외) 한 값, 카드 분리는 1px 테두리(그림자 미사용).
- **애니메이션**: 원판 회전(5.5s)·모달(0.3s fade+scale)·컨페티만. 그 외 장식 애니메이션 금지.
- **이모지**: 모달 당첨 표시 1개만(헤더 0).

### 4.3 레이아웃
- 데스크톱(≥920px): `grid 1fr 360px` (원판 카드 | 사이드).
- 모바일(<920px): 1열(원판 → 사이드). 원판 `aspect-ratio 1/1`로 잘리지 않음.

## 6. 검증 체크리스트 (구현 후 한 줄씩 확인)

### 6.1 기능
- [ ] 인원수/경품 0이면 SPIN 비활성
- [ ] 상품명 빈값·갯수 0/빈값이면 추가 안 됨(shake), 갯수 1000→999, 동일명 병합
- [ ] × 삭제, 추가 직후 원판 갱신, count 0 경품 회색 표시
- [ ] SPIN → **번호 공개(~0.8s) → 원판 회전(5~6s) → 정지**
- [ ] 같은 번호 재추첨 안 됨, 모든 번호 소진 시 SPIN 비활성
- [ ] **원판 정지 슬라이스 = 결과 패널 = 모달 = 기록 (일치)** ⭐
- [ ] 당첨 시 count 1 감소, 소진 칸 재배정(없으면 꽝), **섹션 수 불변**
- [ ] 당첨만 컨페티 발사·자동 소멸(꽝은 없음)
- [ ] 통계(남은 경품/추첨/남은 인원) 실시간, 기록 최신이 위
- [ ] 초기화 확인 다이얼로그 + 즉시 0도 복귀, 예시 불러오기 동작

### 6.2 디자인 (AI 흔적 0)
- [ ] 배경 단색, `backdrop-filter`/그라데이션/`background-clip:text` 없음
- [ ] 원판 2색 교차(+꽝 회색), 액센트 1색(빨강), 컨페티 흑·회색·빨강
- [ ] 디스플레이 폰트 없음, 다중 `box-shadow` 없음, `border-radius` 6px/원형
- [ ] 이모지 2~3개 이하, 호버 translateY 없음, 배경 파티클 없음

### 6.3 코드
- [ ] 단일 HTML 파일, 외부 CSS/JS/이미지/CDN 없음, 시스템 폰트로 표시
- [ ] `node --check` 통과, `var` 0건, 매직 넘버 상수화, `escapeHtml` 적용
- [ ] 모바일 360px에서 원판 안 잘림, `prefers-reduced-motion` 적용

### 6.4 접근성
- [ ] Tab만으로 모든 기능, Enter로 추가·SPIN
- [ ] 결과·번호 영역 `aria-live`, 모달 `role=dialog`·Esc·포커스 관리
- [ ] 주요 버튼 ≥44px, 텍스트 대비 4.5:1
