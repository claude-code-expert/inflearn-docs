# PRD — Lucky Draw 2단계 추첨 사이트

> **작업 범위**: 단일 HTML 파일(`lucky-draw.html`)에 HTML + CSS + JavaScript를 모두 포함한 정적 럭키 드로우 사이트를 구현합니다.
> 외부 빌드 도구·프레임워크·패키지 매니저·CDN을 사용하지 않습니다.
> **문서 역할 분리** — PRD(왜·무엇) · `TRD.md`(어떻게: 데이터·알고리즘) · `REQUIREMENTS.md`(기능 명세·UI·검증) · `ANALYSIS.md`(확정 결정·변경 이력). 네 문서는 서로 중복 없이 참조합니다.

---

## 1. 제품 개요

### 1.1 한 줄 정의
인원 번호를 **랜덤으로 즉시 추첨**해 당첨자를 정한 뒤, **경품 원판을 돌려** 그 사람이 받을 상품을 정하는 **2단계 럭키 드로우** 단일 페이지.

### 1.2 배경
오프라인 행사·온라인 세미나·팀 회식에서 즉석으로 경품을 추첨할 때, 종이 제비뽑기·엑셀 random·외부 추첨 사이트는 신뢰성·재미·접근성이 부족하다. 회전 원판은 **돌아가는 과정 자체가 콘텐츠**이고, 멈추는 위치를 눈으로 따라가며 검증할 수 있어 신뢰감을 준다. 여기에 **"누가 받을지(번호) → 무엇을 받을지(경품)"** 2단계를 더해 행사 진행의 긴장감을 만든다.

### 1.3 목표
- 첫 방문자가 **30초 안에** 인원·경품을 입력하고 첫 추첨을 완료한다.
- 한 화면에서 **입력 → 번호 추첨 → 경품 원판 → 결과 → 다음 추첨**이 끊김 없이 돈다.
- 결과가 **눈으로 검증 가능**(원판이 멈춘 칸 = 당첨 경품)하여 신뢰감을 준다.

## 2. 대상 사용자

| 사용자 유형 | 사용 맥락 | 핵심 니즈 |
|---|---|---|
| 사내 행사 진행자 | 분기 회의·송년회·팀 빌딩 | 빠른 세팅, 큰 화면, 다회 추첨 |
| 온라인 세미나 호스트 | 화상회의 화면 공유 | 시각적으로 명확한 결과, 사운드 불필요 |
| 매장 운영자 | 고객 이벤트, SNS 라이브 | 신뢰감 있는 결과, 모바일·태블릿 대응 |
| 학생·일반 사용자 | 친구 모임, 게임 | 무료, 가입 불필요, 즉시 사용 |

**핵심 페르소나**: "행사 시작 5분 전에 경품 추첨 도구가 필요해진 진행자"

## 3. 핵심 사용 시나리오 (2단계 플로우)

### 시나리오 A — 즉석 추첨 (가장 빈번)
1. 진행자가 URL을 열거나 HTML 파일을 더블클릭한다.
2. **인원수**(예: 10)를 입력한다 → 추첨 대상 번호는 1~10이 된다.
3. **경품**을 등록한다(예: 키보드 2, 마우스 3, 스타벅스 만원·3만원·5만원 각 1) → 원판에 경품 슬라이스 + **꽝 1칸**이 만들어진다.
4. 중앙의 **SPIN** 버튼을 누른다.
   - 먼저 **당첨 번호**가 0.8초 만에 랜덤으로 공개된다(예: "당첨 번호 3번").
   - 이어서 **경품 원판**이 5~6초간 회전 후 멈춘다.
5. 멈춘 칸의 경품(또는 꽝)이 결과 패널·모달·기록에 동시에 표시된다.
6. 진행자가 3번 참가자에게 경품을 전달한 뒤, 다시 SPIN을 눌러 **다음 번호**를 추첨한다.

### 시나리오 B — 빠른 체험 (첫 방문)
1. 처음 방문한 사용자가 **예시** 버튼을 눌러 인원 10·경품 4종을 즉시 채운다.
2. SPIN을 눌러 번호 추첨 → 원판 회전 동작을 확인한다.

### 시나리오 C — 다회 연속 추첨 + 재고 재배정
1. 인원 10, 경품을 여러 종 등록하고 SPIN을 연속으로 누른다.
2. **같은 번호는 다시 뽑히지 않는다**(1인 1회). 매 회 당첨된 경품의 갯수가 1씩 차감된다.
3. 갯수가 0이 된 경품의 원판 칸은 **사라지지 않고**, 재고가 남은 다른 경품으로 **재배정**된다(원판 칸 수는 항상 일정).
4. 모든 경품이 소진되면 남은 칸은 **꽝**으로 채워지고, 남은 인원은 꽝으로 추첨된다.
5. 모든 번호(N명)를 다 뽑으면 SPIN이 비활성화된다.

## 4. 핵심 가치 제안
1. **단일 파일 / 제로 백엔드** — 인터넷 없이도 동작하는 정적 페이지.
2. **2단계 추첨** — "누가(번호) → 무엇을(경품)"으로 행사 긴장감 ↑.
3. **시각적 신뢰성** — 원판이 멈춘 칸이 그대로 결과의 근거.
4. **재고 재배정 + 꽝** — 경품이 떨어져도 원판이 비지 않고, 꽝으로 끝까지 재미를 유지.

## 6. 범위 (Scope)

### 6.1 In-Scope
- **인원수 입력**(번호 풀 1~N) · **경품 추가·삭제·동일명 병합·갯수 입력**
- **번호 추첨**(균등 랜덤, 중복 없음, 즉시 공개) → **경품 원판 회전**(고정 섹션 + 꽝)
- 결과 3중 표시(패널 + 모달 + 기록) · 당첨 시 컨페티
- **재고 차감 + 재배정**(소진 칸을 다른 경품으로, 없으면 꽝) · 통계(남은 경품/추첨/남은 인원)
- 전체 초기화 · 예시 불러오기

### 6.2 Out-of-Scope
- 계정·로그인 · 서버 저장·공유 링크 · 사운드 · 다국어 · 광고/추적
- 다크 모드 · localStorage 영속화(새로고침 시 초기 상태) · 휠 외 추첨 방식
- 결과 서명·해시 검증 · 확률 가중치 직접 입력

## 7. 디자인 원칙 (AI 흔적 배제 — 상세는 `REQUIREMENTS.md` §4)
1. **억제된 팔레트** — 흑/백 + **단일 액센트(빨강)**. 보조색 없음(회색 명도로 계층).
2. **물리적 사물의 인상** — 원판은 평면 색면 분할(2색 교차), 디지털 그라데이션 금지.
3. **장식 절제** — backdrop blur·다중 그림자·별빛 배경 등 분위기 효과 금지.
4. **타이포 일관성** — 시스템 sans 하나(디스플레이 폰트 금지).
5. **모서리·테두리 절제** — `border-radius`는 6px(원형 제외) 한 값, 테두리 1px.
