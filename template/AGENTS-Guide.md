# AI 에이전트 지침 파일(AGENTS.md) 작성 가이드

---

## 🤖 AI 지침 파일이란?

AI 지침 파일(흔히 `AGENTS.md`, `CLAUDE.md`, 또는 `.cursorrules`로 명명)은 **AI 에이전트가 프로젝트의 문맥을 빠르게 파악하고 일관된 코드를 작성하게 만드는 가이드북**입니다.

### 핵심 역할
| 파일 | 대상 | 목적 |
|------|------|------|
| **README.md** | 인간 개발자 | 프로젝트 소개, 설치 방법, 기여 방법 |
| **AGENTS.md** | AI 에이전트 | 코드 컨벤션, 빌드/테스트 명령어, 금지 패턴 |

### 도구별 권장 파일명
- **Claude Code**: `CLAUDE.md` (공식 권장)
- **Cursor**: `.cursorrules`
- **Windsurf**: `.windsurfrules`
- **공통/커스텀**: `AGENTS.md` (에이전트에게 이 파일을 먼저 읽으라고 지시하여 사용 가능)

## 🎯 핵심 원칙

### 1. 실행 가능한(Executable) 정보 중심
AI는 추상적인 철학보다 당장 터미널에 입력할 **명령어**와 준수해야 할 **타입 정의**를 더 잘 이해합니다.

### 2. 토큰 효율성 (Conciseness)
이 파일은 모든 대화의 컨텍스트에 포함될 수 있습니다. 너무 길면 비용이 증가하고 AI의 주의력이 분산됩니다. **핵심 규칙 위주로 작성**하고 상세 문서는 링크로 처리하세요.

### 3. "Do" 보다는 "Don't" (Constraints)
AI가 가장 실수하기 쉬운 부분이나 보안상 위험한 패턴을 **금지 사항**으로 명확히 규정하는 것이 효과적입니다.

## 📋 필수 섹션 구성

### 1. 프로젝트 개요 (Project Context)
프로젝트의 목적과 핵심 아키텍처를 2~3문장으로 설명합니다.
> **예시:** "Next.js 14 기반의 이슈 트래킹 시스템(Tika). App Router와 Server Actions를 사용하며 스타일링은 Tailwind CSS를 채택함."

### 2. 프로젝트 구조 (Structure)
핵심 로직이 어디에 있는지 명시합니다. AI가 파일을 생성할 위치를 헷갈리지 않게 합니다.
- `src/components/ui`: shadcn/ui 기반 공통 컴포넌트
- `src/lib/api`: API 통신 및 데이터 페칭 로직
- `src/store`: 전역 상태 관리 (Zustand)

### 3. 주요 명령어 (Commands)
AI가 스스로 테스트를 돌리거나 린트를 잡을 때 사용합니다. **문서 상단에 배치**하는 것이 좋습니다.
- **개발 서버:** `npm run dev`
- **테스트 실행:** `npm test`
- **린트 수정:** `npm run lint:fix`

### 4. 코딩 컨벤션 (Conventions)
프로젝트 특유의 스타일을 정의합니다.
- **네이밍:** 변수/함수는 `camelCase`, 컴포넌트는 `PascalCase`.
- **패턴:** API 호출 시 `try/catch` 대신 `Result` 객체 패턴 사용.
- **UI:** 모든 UI 컴포넌트는 `@/components/ui`에 있는 것만 재사용.

### 5. 🚨 금지 사항 (Critical Constraints)
- `force push` 금지.
- 환경 변수(`.env`)를 코드에 직접 하드코딩 금지.
- `any` 타입 사용 지양 (반드시 인터페이스 정의).
- 사용자 승인 없이 데이터베이스 스키마 수정 금지.
