00터미널이란? · 화면 읽는 법난이도 쉬움

마우스 없이 키보드로 컴퓨터에 명령을 내리는 애플리케이션입니다. 
평소 Finder(macOS)나 탐색기(Windows)에서 폴더를 더블클릭하던 일을, 여기서는 글자로 입력합니다.

왜 명령어가 이렇게 짧을까?

1970년대 Unix 시절엔 글자 하나하나가 종이에 인쇄되는 단말기를 썼습니다.
출력이 느려서 명령을 최대한 줄였죠. 그래서 list→ls, copy→cp, remove→rm이 된 겁니다.
영어 단어의 약어라는 걸 알면 훨씬 친근해집니다.

터미널 여는 법

OS방법

WindowsWin+R → wt 또는 powershell (예제 따라하려면 Git Bash 혹은 WSL2 권장)
LinuxCtrl+Alt+T

Windows 사용자 팁

이 문서의 예제는 macOS·Linux 셸 기준입니다. Windows에서 똑같이 따라 하려면, Git 설치 시 함께 깔리는 Git Bash 혹은 wsl --install 을 설치하여 실습합니다 (PowerShell은 touch·grep 등 일부 명령이 다릅니다.)

01첫걸음 — 위치 보고 돌아다니기난이도 쉬움

윈도우의 경우 WSL 먼저 설치 후 다음의 명령어들을 실습합니다.

# (요약) PowerShell 관리자로 실행 
wsl --install                # WSL2 + Ubuntu, 재부팅
wsl -l -v                    # 설치된 배포판과 VERSION(2여야 함) 확인
wsl --set-default-version 2  # (혹시 1이면) 기본 버전을 2로


명령약어뜻

lslist현재 폴더 안 목록 보기

$ pwd
/Users/codevillain
$ ls

cd ~ 또는 cd홈 폴더로
cd -직전에 있던 폴더로 (뒤로가기)

가장 강력한 시간 절약 — Tab 자동완성

이름을 다 칠 필요 없습니다. 앞 글자만 치고 Tab을 누르면 나머지가 자동으로 완성됩니다. 오타도 막아 줍니다.

$ cd Doc[Tab]      # → cd Documents/ 로 자동 완성
$ cd D[Tab][Tab]   # → D로 시작하는 후보를 전부 보여줌

알아두면 편한 3가지

$ clear            # 화면 깨끗이 비우기
# ↑ (위 화살표)    → 직전에 친 명령 다시 불러오기
$ open .           # 현재 폴더를 GUI로 열기 (Windows: start . / Linux: xdg-open .)
경로 입력이 귀찮을 때

파일이나 폴더를 터미널 창으로 끌어다 놓으면 전체 경로가 자동으로 입력됩니다. 긴 경로를 손으로 칠 필요가 없습니다.

02만들기 — 폴더와 파일난이도 쉬움

명령약어의미
mkdirmake directory새 폴더 만들기
touch"건드리다"(영단어)빈 파일 만들기
echo"메아리"(영단어)글자를 화면에 출력
 
$ mkdir my-project           # my-project 폴더 만들기
$ mkdir -p a/b/c             # 중간 폴더까지 한 번에 (-p = parents)

$ echo "Hello"               # 화면에 Hello 출력

03보기 — 파일 내용 출력난이도 기본

명령약어 / 어원의미
catconcatenate(이어붙이기)파일 내용 전체 출력
less"less is more" 농담긴 파일을 페이지 단위로 (q로 종료)
head"머리"파일 앞부분
tail"꼬리"파일 뒷부분
 
$ cat README.md        # 전체 출력
$ less long-file.txt   # 페이지 넘기며 보기 (q로 종료)
$ head -5 file.txt     # 앞 5줄만
$ tail -5 file.txt     # 뒤 5줄만
$ tail -f server.log   # 실시간으로 따라보기 (-f = follow)
재미있는 어원

cat은 고양이가 아니라 concatenate(연결)의 약어입니다. 원래 여러 파일을 이어 붙이는 용도였는데, 한 파일만 넣어도 작동해서 "내용 보기"로 더 자주 쓰입니다. less는 옛 more 명령의 후속이라 "less is more" 격언을 패러디한 이름입니다.

04옮기고 지우기 (rm 주의)난이도 기본

명령약어의미
cpcopy파일 복사
mvmove파일 이동 또는 이름 변경
rmremove파일 삭제
 
$ cp README.md README-backup.md   # 복사
$ cp -r my-project backup-project  # 폴더 복사 (-r = recursive)
$ mv old.txt new.txt               # 이름 변경
$ mv hello.txt Documents/          # 폴더로 이동
$ rm hello.txt                     # 파일 삭제
$ rm -r my-project                 # 폴더 통째로 삭제
절대 주의 — rm은 휴지통을 거치지 않습니다

터미널의 rm은 한 번 지우면 끝, 복구가 어렵습니다. 특히 rm -rf /는 "루트부터 강제로 전부 지워라"라는 뜻으로 컴퓨터를 통째로 날립니다. 절대 따라 치지 마세요. 안전장치로 다음 줄을 ~/.zshrc에 추가하면 rm 때마다 한 번씩 물어봅니다: alias rm='rm -i' (적용이 안되면 터미널 재실행 필요)

05옵션(플래그) 이해하기난이도 기본

명령 뒤에 붙는 -r, -f 같은 것이 옵션(플래그)입니다. 이것도 전부 영어 약어입니다.

옵션약어의미
-r / -Rrecursive폴더 안쪽까지
-fforce묻지 말고 강제로
-aall숨김 파일 포함 모두
-llong자세히(긴 형식)
-vverbose진행 상황 출력
-iinteractive작업 전에 물어보기
-pparents중간 폴더까지 만들기

옵션은 합쳐 쓸 수 있고, 순서는 상관없습니다.

$ ls -l -a     # 와
$ ls -la       # 와
$ ls -al       # 모두 같음

왜 rm -rf가 위험한지 분해해 보면

rm   -r   -f
 │    │    │
 │    │    └─ force      : 묻지 말고
 │    └────── recursive  : 폴더 안쪽까지
 └─────────── remove     : 삭제하라

→ "묻지 말고 폴더 안쪽 전부 지워라"  (그래서 위험)

06와일드카드와 경로 기호난이도 기본+

와일드카드 — 여러 파일을 한 번에

*는 "아무 글자나 여러 개", ?는 "아무 글자 한 개"를 뜻합니다.

$ ls *.md            # .md로 끝나는 파일 모두
$ ls report-?.txt    # report-1.txt, report-a.txt 처럼 한 글자만 다른 것
$ cp *.jpg Pictures/ # 모든 jpg를 Pictures 폴더로 복사
주의

rm *.tmp처럼 와일드카드를 삭제에 쓸 때는 한 번 더 확인하세요. 먼저 ls *.tmp로 지워질 목록을 눈으로 본 뒤 삭제하는 습관이 안전합니다.

경로 기호 정리

기호의미
.현재 폴더 (git add .의 .도 "지금 이 폴더 전체")
..한 단계 위(부모) 폴더
~내 홈 폴더 (/Users/사용자이름)
/최상위(루트) 폴더 — 시작점

절대경로는 /로 시작하는 "어디서든 통하는 전체 주소"(/Users/me/Documents), 상대경로는 지금 위치 기준 주소(../Pictures)입니다.

07찾기와 도움말난이도 중급

명령약어 / 어원의미
manmanual명령어의 전체 설명서 (q로 종료)
which"어느 것?"명령어가 설치된 위치
grepg/re/p파일 안에서 패턴(글자) 검색
find"찾다"파일 찾기
 
$ man ls                  # ls의 전체 매뉴얼 (q로 종료)
$ which python            # python이 설치된 경로
$ grep "error" server.log # server.log에서 "error"가 들어간 줄 찾기
$ find . -name "*.md"     # 현재 폴더 아래 모든 .md 파일
grep 어원

옛 편집기 ed의 명령 g/re/p(global / regular expression / print = 전체에서 정규식으로 찾아 출력)에서 왔습니다. 너무 유용해서 별도 명령으로 독립한 것입니다.

08묶어 쓰기 — 파이프·리다이렉션난이도 중급

명령 하나의 결과를 다음 명령에 넘기거나(파이프), 파일로 저장(리다이렉션)할 수 있습니다. 여기서부터 터미널이 강력해집니다.

기호이름의미
|파이프앞 명령의 결과를 뒤 명령에 넘김
>리다이렉션출력을 파일로 저장(덮어쓰기)
>>추가 리다이렉션파일 끝에 이어붙이기
 
$ ls -l | grep ".md"        # ls 결과를 grep으로 넘겨 .md 줄만 보기
$ history | grep git        # 지난 명령 중 git이 들어간 것만
$ echo "첫 줄" > note.txt     # note.txt에 저장 (기존 내용 덮어씀)
$ echo "둘째 줄" >> note.txt   # note.txt 끝에 이어붙임
기억할 차이

>는 기존 내용을 전부 지우고 새로 씁니다. 기존 내용을 살리고 싶으면 반드시 >>를 쓰세요.

09관리자 권한 — sudo난이도 중급

sudo는 SuperUser DO의 약어로 "최고 권한자로 실행"이라는 뜻입니다. 비밀번호를 물어보는 명령은 거의 다 sudo입니다.

$ sudo apt install gh   # 관리자 권한으로 프로그램 설치 (Linux)
신중하게

sudo는 시스템 전체에 영향을 주는 명령이라는 신호입니다. 의미를 모르는 sudo 명령은 인터넷에서 복사해 그대로 실행하지 마세요.

12편집기 ② vi / vim (탈출법)난이도 중급

거의 모든 Unix 계열에 기본 설치된 편집기입니다. (vi = visual editor, 요즘 실행되는 건 보통 vim = vi improved.) nano보다 어렵지만, 최소한 빠져나오는 법은 꼭 알아야 합니다.

가장 먼저 외울 것 — 탈출법

vi에 갇혔다면: Esc → :q! → Enter ("Escape, 콜론, q, 느낌표, 엔터" = 저장 안 하고 강제 종료). 이것만 알아도 패닉에서 벗어납니다.

핵심 개념 — 모드(Mode)

vi는 "글 쓰는 모드"와 "명령 내리는 모드"가 따로 있습니다. 일반 모드는 리모컨, 입력 모드는 펜이라고 생각하세요. 글을 쓰려면 펜으로 바꾸고(i), 다 쓰면 다시 리모컨으로 돌아옵니다(Esc).

이게 vi의 80% — 절대 외울 흐름

1. vi 파일명     ← 파일 열기
2. i            ← 입력 모드로 (글자 쓸 수 있음)
3. (글자 입력)
4. Esc          ← 일반 모드로 복귀
5. :wq          ← 저장하고 종료 (Write + Quit)

저장·종료 명령 (Esc 후)

명령약어의미
:wwrite저장
:qquit종료
:wqwrite + quit저장하고 종료
:q!quit + 강제저장 안 하고 강제 종료
비개발자가 가장 자주 만나는 상황

git commit을 했더니 vi가 떴다면: i → 메시지 입력 → Esc → :wq → Enter. 망쳤으면 Esc → :q!로 나가서 다시 시작하세요. 이동은 hjkl 대신 화살표 키를 그냥 쓰면 됩니다.

13실전 예제 모음 (자주 하는 작업)난이도 실무

앞에서 배운 명령들을 "이럴 땐 이렇게" 형태로 묶었습니다. 상황별로 통째로 복사해 쓰고, 익숙해지면 한 줄씩 바꿔 보세요. 아래 code는 VS Code를 여는 명령이며, 없으면 nano로 바꿔도 됩니다.

① 셸 설정 파일(.zshrc / .bashrc) 편집 후 적용

환경변수·별칭(alias)·PATH를 추가할 때 가장 많이 합니다. 저장해도 새 터미널을 열거나 source로 다시 읽어야 적용됩니다.

$ nano ~/.zshrc            # 설정 파일 열기 (zsh 기본). bash라면 ~/.bashrc
# 예: 맨 아래에 한 줄 추가
#   alias gs='git status'
# Ctrl+O → Enter 저장 → Ctrl+X 종료
$ source ~/.zshrc          # 지금 창에 바로 적용 (창 새로 열기와 동일)
$ gs                       # 방금 만든 별칭이 동작하는지 확인
내 셸이 zsh인지 bash인지 모를 때

echo $SHELL을 치면 알려줍니다. 끝이 /zsh면 ~/.zshrc, /bash면 ~/.bashrc(또는 ~/.bash_profile)를 편집하세요. macOS 최신 기본은 zsh입니다.

② 빈 파일 만들고 바로 편집하기

$ touch script.sh          # 빈 파일 생성
$ nano script.sh           # 편집기로 열어 내용 작성 (또는 code script.sh)
# --- 한 번에 만들면서 첫 줄까지 넣기 ---
$ echo '#!/bin/bash' > run.sh   # 새 파일에 첫 줄 작성(덮어쓰기)
$ echo 'echo 안녕' >> run.sh    # 다음 줄 이어붙이기

③ 권한 주기 — chmod 755 / 644

스크립트를 실행 가능하게 만들 때 chmod(change mode)를 씁니다. 숫자 3자리는 순서대로 소유자 / 그룹 / 그 외 모든 사용자의 권한이고, 각 자리는 읽기(4)+쓰기(2)+실행(1)을 더한 값입니다.

숫자권한(rwx)주로 쓰는 곳
7 = 4+2+1읽기·쓰기·실행소유자에게 모든 권한
6 = 4+2읽기·쓰기일반 문서 파일
5 = 4+1읽기·실행실행은 되지만 수정은 금지
4 = 4읽기만읽기 전용
 
$ chmod 755 script.sh      # 나=rwx, 그룹=r-x, 그외=r-x  (실행 파일·스크립트 표준)
$ chmod 644 notes.txt      # 나=rw-, 그룹=r--, 그외=r--  (일반 문서 표준)
$ chmod +x script.sh       # 숫자 대신: "실행 권한만 추가" (간편)
$ ls -l script.sh          # 확인 → -rwxr-xr-x 로 보이면 성공
$ ./script.sh              # 실행 (앞의 ./ = "지금 폴더의 이 파일")
755가 -rwxr-xr-x로 보이는 이유

7=rwx, 5=r-x, 5=r-x. 맨 앞 -는 "파일"(폴더면 d)을 뜻합니다. 권한이 없으면 그 자리에 -가 찍힙니다.

④ 파일 복사하기

$ cp report.txt report-backup.txt   # 같은 폴더에 사본 만들기
$ cp report.txt ~/Documents/         # 다른 폴더로 복사 (이름 유지)
$ cp -r src/ src-backup/             # 폴더 통째로 복사 (-r = 안쪽까지)

⑤ 파일·폴더 이동하기 / 이름 바꾸기

mv는 이동과 이름 변경이 같은 명령입니다. "옮긴다"가 곧 "이름을 바꾼다"입니다.

$ mv draft.txt final.txt       # 이름 변경 (draft → final)
$ mv final.txt ~/Documents/     # Documents 폴더로 이동
$ mv *.png images/              # 모든 png를 images 폴더로 한 번에 이동

⑥ 파일 삭제하기

$ rm temp.log               # 파일 1개 삭제
$ rm a.txt b.txt c.txt      # 여러 개 한 번에
$ rm -i important.txt       # 지우기 전에 물어보게 (-i = 안전)
다시 강조 — rm은 휴지통이 없습니다

지우기 전에 ls로 한 번 확인하는 습관이 가장 안전합니다. 와일드카드(rm *.log)는 특히 신중하게. (자세한 내용은 04 옮기고 지우기 참고)

⑦ 디렉토리(폴더) 삭제하기

$ rmdir empty-folder        # 비어 있는 폴더만 삭제 (안전)
$ rm -r old-project         # 안에 내용이 있어도 통째로 삭제
$ rm -ri old-project        # 통째로 삭제하되 항목마다 물어보기 (더 안전)

⑧ 파일이 있는 폴더를 Finder/탐색기로 열기

터미널 위치를 그대로 GUI 창으로 여는 가장 편한 방법입니다.

$ open .                    # 현재 폴더를 Finder로 열기 (macOS)
$ open ~/Downloads          # 특정 폴더 열기
$ open report.pdf           # 파일을 기본 앱으로 바로 열기
# Windows(Git Bash): start .   /   Linux: xdg-open .

⑨ 스크립트 작성 → 권한 → 실행, 전체 흐름

위 작업들을 하나로 이으면 실제 개발 작업이 됩니다.

$ touch hello.sh                 # 1. 빈 파일 생성
$ nano hello.sh                  # 2. 내용 작성 후 저장·종료
#    #!/bin/bash
#    echo "Hello, Terminal!"
$ chmod +x hello.sh              # 3. 실행 권한 부여
$ ./hello.sh                     # 4. 실행 → Hello, Terminal! 출력

A. 어원으로 외우는 핵심 명령

pwd   = Print Working Directory   → 현재 위치
ls    = list                      → 목록

-f    = force       → 묻지 말고

-l    = long        → 자세히
-v    = verbose     → 진행 상황 출력
-i    = interactive → 물어보면서
-p    = parents     → 중간 폴더까지
